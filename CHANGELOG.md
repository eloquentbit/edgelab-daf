## Versioning

This project follows Semantic Versioning.

- **MAJOR** version changes introduce incompatible architectural or API changes.
- **MINOR** version changes introduce backward-compatible functionality.
- **PATCH** version changes include bug fixes, documentation improvements and minor internal changes.
